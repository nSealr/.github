# License Policy

nSealr uses maximum practical permissiveness.

## Defaults

- Software, firmware, tooling, and website code: MIT.
- Specs, schemas, examples, test vectors, lab docs, website content, and general
  documentation: CC0-1.0.
- Hardware designs, PCB files, BOMs, enclosure files, and assembly artifacts:
  CERN-OHL-P-2.0.

## Requirements

- Every repository must include a `LICENSE` file.
- Every README must explain the repository-specific license policy.
- Imported or forked code must keep its upstream license and attribution.
- License-incompatible code must not be mixed into MIT/CC0/CERN-OHL-P work.
